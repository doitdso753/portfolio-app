쇼핑몰통합관리 서비스

담당업무

  • React 쇼핑몰통합관리 화면 구성과 컴포넌트 설계 및 개발
  • Node.js 상품, 주문, 배송 API 설계 및 개발
  • 쇼핑몰 Open API 분석 및 상품, 주문, 배송 프로세스 연동
  • Next.js 백오피스 관리자 기능 API 설계 및 개발
  • React 백오피스 화면 구성과 컴포넌트 설계 및 개발

기술스택

  • Frontend: React, Redux, Next.js, JavaScript, Storybook, styled-components, MUI, ESLint, Prettier
  • Backend: Node.js, Swagger UI, Puppeteer
  • Database: MariaDB
  • Tools/Infra: Docker, Linux, Git, GitLab, Jira, Confluence, IntelliJ

주요 경험 - 서비스

마켓별 상이한 데이터 구조를 서비스 공통 모델로 변환하여 기존 서비스 구조를 유지하면서 신규 판매 채널을 연동했습니다. Open API에서 제공하지 않는 데이터는 Puppeteer를 활용해 수집하여 서비스에서 필요한 데이터를 처리할 수 있도록 구현했습니다. Atomic Design과 Storybook을 적용해 공통 컴포넌트를 구성하고 마켓 정책을 Configuration으로 분리하여 신규 마켓 추가 시 컴포넌트 수정 없이 확장할 수 있는 구조를 설계했습니다.

주요 경험 - 백오피스

운영 과정에서는 개발자가 직접 DB를 수정해야 하는 반복 업무를 개선하기 위해 Next.js 기반 백오피스를 구축했습니다. CRUD 화면을 공통 컴포넌트로 설계하고 Redux와 SWR을 역할에 맞게 분리하여 상태 관리를 구성했습니다. 택배사, 카테고리 매핑 등 기초 데이터 관리 기능을 구현해 운영팀이 직접 데이터를 관리할 수 있는 환경을 마련했습니다. 대량 데이터 등록을 위한 엑셀 일괄 등록 기능과 데이터 변경 이력을 확인할 수 있는 액션 로그 기능을 개발해 운영 효율을 높였습니다.

시스템 구성도

UI Service(React)와 Backend/Core Service(Node.js)로 구성된 쇼핑몰 통합관리 서비스입니다.
사용자는 UI Service를 통해 상품, 주문, 배송 관리 기능을 이용하며, UI Bridge가 Backend Service와 연동하여 비즈니스 기능을 제공합니다. Backend Service는 도메인별 API로 구성해 기능별 책임을 분리하고 Kafka 메시지 큐로 Backend Service 작업을 비동기로 처리합니다. Next.js 기반 Admin Service에서 마켓 기초 데이터 등 운영 기능을 관리할 수 있도록 구현했습니다. Docker/Kubernetes 컨테이너 환경에서 서비스를 운영했습니다.
